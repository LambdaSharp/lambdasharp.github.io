Config Command Line

The config command is used to configure the λ# CLI. The configuration step optionally creates resources for deploying λ# modules and captures deployment preferences. The λ# CLI configuration options are stored in AWS System Manager Parameter Store, so they can be shared across teams on the same AWS account.

Options

--existing-s3-bucket-name <NAME>

(optional) Existing S3 bucket name for module deployments (blank value creates new bucket)

--requested-s3-bucket-name <NAME>

(optional) Requested S3 bucket name for module deployments (blank value assigns automatic name)

--cloudformation-notifications-topic <ARN>

(optional) Existing SNS topic ARN for CloudFormation notifications (blank value creates new topic)

--protect

(optional) Enable termination protection for the CloudFormation stack

--force-update

(optional) Force CLI profile update

--cli-profile|-C <NAME>

(optional) Use a specific LambdaSharp CLI profile (default: Default)

--aws-profile|-P <NAME>

(optional) Use a specific AWS profile from the AWS credentials file

--verbose|-V:<LEVEL>

(optional) Show verbose output (0=quiet, 1=normal, 2=detailed, 3=exceptions)

Configure a new CLI profile without prompts

Using Bash:

lash config \
    --cli-profile Team \
    --existing-s3-bucket-name "" \
    --requested-s3-bucket-name "" \
    --cloudformation-notifications-topic ""

Using PowerShell:

lash config ^
    --cli-profile Team ^
    --existing-s3-bucket-name "" ^
    --requested-s3-bucket-name "" ^
    --cloudformation-notifications-topic ""
